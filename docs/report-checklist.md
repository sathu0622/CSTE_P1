# CTSE Assignment Report Checklist

Use this checklist to ensure your report and demo align with assignment requirements.

## Required Report Sections

- [ ] Shared architecture diagram with all 4 services and communication paths
- [ ] Description of your assigned microservice and its role
- [ ] Evidence of communication with another member's service
- [ ] DevOps practices used (CI/CD, Docker, registry)
- [ ] Security practices used (JWT, validation, rate limit, helmet, least privilege)
- [ ] DevSecOps tool usage (Snyk/SonarCloud output or screenshots)
- [ ] Challenges and how they were resolved

## Code Repository Checklist

- [ ] Public repository link included
- [ ] Swagger/OpenAPI for each microservice
- [ ] Dockerfile per service
- [ ] CI/CD workflow files included
- [ ] Deployment workflow/template included

## Demo Checklist (10 minutes)

- [ ] Show deployed service endpoint on cloud
- [ ] Show one inter-service call with live output
- [ ] Show one cross-member integration call
- [ ] Show CI pipeline run for a commit
- [ ] Show Snyk/SonarCloud scan result
- [ ] Explain security controls in the running system
